# Ho-10.1-AT-02 — The tool: `ZFSMutationTool` fills the seam

**Goal**

Fill the Workbench mutation seam ho-10 left named-and-refused: a `ZFSMutationTool` in `PalanaCore` declaring the eight mutation verbs (`.mutation` kind, `.zfs` requirement) and composing a `PlanRequest` from a verb and its gathered input. The seam itself reshapes to carry the gather. Pure `PalanaCore` — no app target, no UI (that is AT-03).

**Context**

ho-10.1 Decisions 1, 5, 6 govern (read `ho-process/hos/ho-10.1-the-zfs-tool.md`). AT-01 built the engine side — read these before writing anything:

- `Sources/PalanaCore/Plan/ZFSMutation.swift` — the payload the tool builds.
- `Sources/PalanaCore/Plan/PlanRequest.swift` — the request shape (`operation: .zfs`, `zfs:` payload, empty `entries`, nil `destination`).
- `Sources/PalanaCore/Workbench/WorkbenchTool.swift` — the vocabulary and the seam as it stands: `planRequest(for:on:)` default-nil in a protocol extension.
- `Sources/PalanaCore/Workbench/SystemReadsTool.swift` — the tool template: struct, verb array, id-switch composition.
- `Sources/PalanaCore/Workbench/Workbench.swift` — the coordinator; **do not change it.** `run` still throws `notARead` for mutations; the mutation route goes app-side in AT-03.
- `Tests/PalanaCoreTests/WorkbenchTests.swift` — the existing battery, including the default-nil `planRequest` test you will update.

**The seam reshape (decided by the session, engineering call under the ho's Decision 5):** the ho-10 hook `planRequest(for verb:on host:)` carries no parameters, but every ZFS mutation needs the target dataset and most need a gathered name or flag. Nothing calls the hook (only the default-nil test), so filling it means shaping it. The parameterless hook is **replaced** — not overloaded — by a parameterized one.

**Required Changes**

1. **`MutationInput`** (in `WorkbenchTool.swift`, beside the verb vocabulary) — what the surface gathers before a mutation verb composes:

   ```swift
   /// What the operator supplied before a mutation verb composes.
   public struct MutationInput: Sendable, Equatable {
       /// The dataset the operator is standing in — the verb's target.
       public var target: String
       /// The gathered name or path, for verbs that need one. nil otherwise.
       public var text: String?
       /// The recursive flag, for verbs that offer it. false otherwise.
       public var recursive: Bool
       public init(target: String, text: String? = nil, recursive: Bool = false)
   }
   ```

2. **`GatherSpec`** (same file) — what a verb needs gathered, so the surface renders the right field without knowing the tool:

   ```swift
   /// What a mutation verb gathers from the operator before composing.
   public struct GatherSpec: Sendable, Equatable {
       /// The field's label — plain sentence, message-grammar voice.
       public var prompt: String
       /// Whether a text field gathers a name or path.
       public var needsText: Bool
       /// Whether the verb offers a recursive choice.
       public var offersRecursive: Bool
       public init(prompt: String, needsText: Bool, offersRecursive: Bool = false)
   }
   ```

   `WorkbenchVerb` gains `public let gather: GatherSpec?`, default `nil` in the init — all existing call sites stand.

3. **The seam** — in the `WorkbenchTool` protocol extension, replace `planRequest(for:on:)` with:

   ```swift
   public func planRequest(
       for verb: WorkbenchVerb, on host: String, input: MutationInput
   ) -> PlanRequest? { nil }
   ```

   Default nil so read tools conform untouched. Update the DocC: the seam is now filled — `ZFSMutationTool` overrides it; the Workbench's read path still refuses mutations (`notARead`), the app routes `.mutation` verbs through this method to the Plan Engine.

4. **`ZFSMutationTool`** — a struct conforming to `WorkbenchTool`, `id: "zfs"`, `label: "zfs"`. `command(for:on:)` is never called for mutations; return `verb.id` (the `SystemReadsTool` default-branch posture). Eight verbs, all `kind: .mutation`, all `requirement: .zfs`:

   | id | label | keyHint | gather |
   |---|---|---|---|
   | `zfs-create` | `new dataset` | `c` | "name the new dataset — a child of <where you stand>", needsText |
   | `zfs-destroy` | `destroy` | `x` | "destroy this dataset", offersRecursive |
   | `zfs-rename` | `rename` | `r` | "type the full new name", needsText |
   | `zfs-snapshot` | `snapshot` | `n` | "name the snapshot", needsText, offersRecursive |
   | `zfs-destroy-snapshot` | `destroy snapshot` | `k` | "name the snapshot to destroy", needsText |
   | `zfs-rollback` | `roll back` | `b` | "name the snapshot to roll back to", needsText |
   | `zfs-set-mountpoint` | `set mountpoint` | `m` | "type the mountpoint path", needsText |
   | `zfs-clear-mountpoint` | `clear mountpoint` | `i` | "clear the mountpoint — the dataset inherits its parent's", no text |

   Prompts are the committed starting copy — AT-03's hands session may reword them; keep them plain sentences, no jargon (Collision.swift's message-grammar header governs).

5. **`planRequest(for:on:input:)`** — switches on `verb.id` and builds the `ZFSMutation`:

   - `zfs-create` → `.createDataset(name: "\(input.target)/\(text)", mountpoint: nil)` — create makes a **child** of the standing dataset (Decision 5). Mountpoint at create stays out; `zfs-set-mountpoint` exists.
   - `zfs-destroy` → `.destroyDataset(name: input.target, recursive: input.recursive)`
   - `zfs-rename` → `.renameDataset(from: input.target, to: text)` — the gathered text is the **full** new dataset name; the surface prefills the field with the current name (AT-03).
   - `zfs-snapshot` → `.snapshot(dataset: input.target, name: text, recursive: input.recursive)`
   - `zfs-destroy-snapshot` → `.destroySnapshot(dataset: input.target, name: text)`
   - `zfs-rollback` → `.rollback(dataset: input.target, name: text)`
   - `zfs-set-mountpoint` → `.setMountpoint(dataset: input.target, path: text)`
   - `zfs-clear-mountpoint` → `.clearMountpoint(dataset: input.target)`

   Where a verb needs text: trim whitespace; nil or empty after trim → return nil (the surface treats an empty submit as a dismissal, ho-9.1's posture; malformed shapes that get through still refuse at the engine's `validateZfs`). Unknown verb id or empty `input.target` → nil.

   The `PlanRequest`: `operation: .zfs`, `source: Locus(host: host, directory: input.target)` — the standing dataset IS the locus; the panel's route line reads `host:tank/data`, which is the truth — `entries: []`, `destination: nil`, `zfs:` the mutation. Token: leave the default.

6. DocC on every public decl, committed-vocabulary voice, matching the tone of `SystemReadsTool.swift` and `WorkbenchTool.swift`.
