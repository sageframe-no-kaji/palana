# Ho-10.4-AT-03 — The recursive toggle gets a key

## Why

The ZFS gather's recursive toggle (`offersRecursive`, e.g. destroy's "destroy
its children and snapshots too — zfs counts both") is currently **mouse-click
only** — the sole setter of `OperationModel.zfsRecursive` is the `.checkbox`
`Toggle` in `PlanPanel`. In a keyboard-first grammar the operator sails past it:
press the verb key, press Enter, and the command composes with recursive still
off. The hands session hit this live — every destroy of a snapshotted dataset
failed with `filesystem has children / use '-r'` because the toggle was never
reachable without the mouse.

## Decision (made — do not re-open)

**Bind `space` to flip `zfsRecursive` while a gather with `offersRecursive == true`
is showing.** NOT `r`: the recursive toggle also appears during the **snapshot**
and **rollback** gathers, which take a typed name — `r` is a legal name
character and would collide with typing. A space is never a legal ZFS
dataset/snapshot name character, so intercepting it as the toggle is safe in
BOTH the field-less gather (destroy) and the text gathers (snapshot, rollback).

## Scope

1. **Key interception** — while the panel is in a gather whose verb offers
   recursive (`operation.pendingZFSVerb?.gather?.offersRecursive == true`),
   `space`:
   - **field-less gather** (destroy, `zfsPaneGatherActive` / `!zfsGatherWantsText`):
     flips `operation.zfsRecursive`, consumes the key (never reaches the panes).
   - **text gather** (snapshot, rollback, `zfsGatherWantsText == true`): flips
     `operation.zfsRecursive` and does NOT insert a space into the text field.
     (Trace the gather's text-field key path; space must be swallowed as the
     toggle, not typed. Since ZFS names forbid spaces, this steals nothing the
     operator could legitimately type.)
   In both cases the flip is a no-op passthrough when the current gather does
   not offer recursive.
   - Trace the field-less key routing from the comment in `PlanPanel` (~line
     253): "Return and Esc are routed through the monitor's field-less branch in
     `PalanaSession.handle(_:)`." Add the space case there for the field-less
     path; add the text-gather path wherever the naming text field consumes keys.

2. **The hint** — the toggle's rendered label (or an adjacent footer line) shows
   the key: prefix or suffix the `toggleLabel` with `[space]` so the operator
   sees `[space]` and the toggle's current on/off state. Keep it terse and in
   the existing panel type styling (`Theme.ink` / size 11). The checkbox stays
   (mouse still works); the key is additive.

## Out of scope

  correct). This AT only adds a keyboard path + hint to the existing toggle.
- The "pane locks until Esc" report — separate, needs a repro; not this task.
