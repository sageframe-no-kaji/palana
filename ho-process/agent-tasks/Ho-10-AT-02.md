# Ho-10-AT-02 — The strip, the focus, the transcript

**Goal**

The read-only tools strip on the terminal's trailing edge, driven by the Workbench that AT-01 built. Subtle buttons—`df`, `zfs list`, `zpool status`, `zpool list`—each gated against the focused pane's host, each dropping raw output into the plan panel's transcript when pressed. Shift-tab is the door into the terminal. The strip disables whenever an operation owns the terminal. Requires Ho-10-AT-01 landed.

**Context**

ho-10 Decision 3: the terminal grows the strip. The strip mounts on the right of the plan panel—`SurfaceView.panes`, the `VSplitView` branch at `if session.operation.panelShowing { PlanPanel(...) }`—wrapped in an `HStack` so the buttons pin to the transcript's trailing edge, not a second surface. The plan panel is the one monospace terminal kamae-2 permits, and round 5 already made it a pane the grammar flows through, toggled by backtick. One rule holds the plan's kamae-2 claim intact: the terminal serves a plan *or* the tools, never both. While `session.operation.active` (gathering, ready, enacting, naming), the strip disables—the transcript belongs to the plan. When idle, the strip is live and its reads accumulate in the transcript as a scrollback; starting an operation resets the transcript exactly as `OperationModel.begin` already does.

Decision 1: a button aims at `session.focusedPane.state.host`. Decision 2: a button is live when `workbench.availability(of:on:)` returns `.available`, disabled with the plain reason otherwise (never silently absent). Decision 4: shift-tab moves focus into the terminal and back; while the terminal holds focus the tool verbs answer their key hints.

The Workbench needs to reach both this Mac and the remote hosts, routed by host name. The enactment already routes this way—`OperationModel.enact` builds `RoutingConduit(remote: engine.conduit)`. Construct the Workbench with the same routing so a `df -h` on `local` runs through the `LocalConduit` and a read on koan runs over ssh. **Confirm `RoutingConduit` routes the `run(on:_:)` read path for the local host; if it only routes the enactment transports, hand the Workbench a conduit that dispatches by `PalanaCore.localHostName` (mirror `Engine.conduit(for:)`).** This is the one seam to verify before building.

Read before writing: `ho-process/agent-tasks/Ho-10-AT-01.md` and the `Sources/PalanaCore/Workbench/` types it created (the real signatures win over this spec if they differ—surface the difference), `SurfaceView.swift` (`panes`, the panel branch, `paneVerb` for the subtle-button idiom, the toolbar), `PlanPanel.swift` (the transcript, `EchoBuffer`, `panelGround`, the header), `OperationModel.swift` (`echo`, `active`, `phase`, `showPanel`/`hidePanel`, `begin`'s transcript reset), `PalanaSession.swift` (`handle`, `focusedPane`, `focusedSide`, the tab/`switchPane` dispatch, the special-key path, the key monitor), `Grammar.swift` (`token`—the shift-tab collision, below).

**Required Changes**

1. **Shift-tab as a distinct token.** In `Grammar.token`, shift-tab currently assembles to `"tab"` (bare shift is dropped for special keys). Add a surgical case: when the special-key base is `tab` and shift is held without cmd or ctrl, return `"shift-tab"`. Touch no other key—shift-arrow, shift-return, and the rest stay as they are.

2. **The Workbench model.** The session builds one `Workbench(conduit: <routing conduit per the context note>, field: engine.field)` and holds the `SystemReadsTool`. A tiny `WorkbenchModel` (app, `@MainActor @Observable`) is fine if it keeps `SurfaceView` clean; folding into `PalanaSession` is also fine—your call, keep the type-body budgets honest.

3. **The strip view.** `WorkbenchStrip`—a vertical column of buttons in the `paneVerb` register (small, `Theme.accent` glyph or short label, `.plain`, `.help(...)`, subtle—no loud chrome, the notebook stays calm). One button per verb, in declared order. Per button:
   - Enabled when `!session.operation.active` **and** `workbench.availability(of: verb, on: focusedHost) == .available`.
   - Disabled otherwise; the disabled reason rides `.help(...)`—the `.unmet` sentence from AT-01 when a capability is missing, nothing needed when the terminal is simply busy (the whole strip dims).
   - **Local honesty (review-caught):** AT-01's `.zfs` evaluation returns "not yet probed" for any host with nil facts, and the local Mac is never probed—so a zfs button on `local` would misread "local not yet probed—probe from the field or map" when the truth is there is no zfs here and no probe to run. When the focused host is `PalanaCore.localHostName`, render the zfs verbs disabled with "no zfs on this Mac" instead. Handle this in the app's gating display; do not reopen AT-01.
   - The label carries the verb's key hint quietly (the menu precedent—`zfs list  z` or a trailing hint), so the keyboard path is legible.
   - Action: `session.runWorkbenchVerb(verb)`.

4. **Mount the strip.** In `SurfaceView.panes`, wrap the panel branch:
   ```swift
   if session.operation.panelShowing {
       HStack(spacing: 0) {
           PlanPanel(operation: session.operation)
           WorkbenchStrip(session: session)
       }
       .frame(minHeight: 130, idealHeight: 280)
   }
   ```
   The strip is narrow and pinned right; the panel keeps the width.

5. **The read run.** `PalanaSession.runWorkbenchVerb(_ verb:)`:
   - Guard `!operation.active`—an operation owns the terminal, the read does not land (belt to the strip's disable).
   - `guard let host = focusedPane.state.host` else return.
   - Guard `workbench.availability(of: verb, on: host) == .available`.
   - `operation.showPanel()` so the terminal is visible.
   - Append a header line and drain the read raw into the transcript, phase untouched. Add the minimal `OperationModel` path for this (below).

6. **The transcript path (`OperationModel`).** Add one method that writes a tool read into `echo` without disturbing the operation phase (stays `.idle`), so the strip stays live for the next read and the terminal reads as a scrollback:
   - `func runToolRead(header: String, stream: RunningCommand) async` — append `── \(header)` as a `.note` line, then drain `stream` raw into `echo` (stdout as `.stdout`, stderr as `.stderr`, using the existing `echo.append`/`appendRaw`/`flushAll` machinery the enactment uses), and `showPanel()` if hidden. No phase change, no `onFinished`, no plan. The header names the command and host: `df -h · koan`.
   - The successive reads accumulate; `begin` already resets `echo` when a real operation starts, so the plan's claim is never blurred.

7. **Terminal focus (shift-tab).** A `PalanaSession.terminalFocused` flag (not persisted):
   - `"shift-tab"` in `handle`: if the terminal is hidden, `operation.showPanel()`; toggle `terminalFocused`. `"tab"` while `terminalFocused` returns focus to the pane (clear the flag) instead of switching panes; `"tab"` otherwise stays `switchPane`.
   - While `terminalFocused`, route keys to the strip: a token matching a verb's key hint runs that verb (gated as the button is); `esc` or `shift-tab`/`tab` clears `terminalFocused` back to the focused pane; the pane grammar (walk, verbs) is suspended so the letters reach the tools. The plan-panel priority keys (backtick hides, etc.) still apply.
   - A quiet focus cue: the strip (or the panel header) brightens while `terminalFocused`, so the operator sees where the keyboard points. Subtle—an accent underline or a slightly stronger ground, not a loud ring.
   - Keyboard scrolling of the transcript under `j`/`k` is a nice-to-have, not required for v1—the transcript auto-follows new output and the trackpad scrolls it. If it falls out cheaply, add it; if not, leave it for a feedback round and note it.

8. **The vocabulary.** `HelpOverlay` gains a line: `⇧�tab` → "into the terminal · tool reads". Nothing else in the card changes.

**Do Not**

- Do not spawn a second terminal or a separate Workbench window. The strip mounts on the existing plan panel—Decision 3, and kamae-2's monospace-appears-once.
- Do not let a read land while an operation is active. Belt (the strip disables) and suspenders (`runWorkbenchVerb` guards `!operation.active`).
- Do not change the operation phase for a tool read. A read is not an operation—`.idle` throughout, no `onFinished`, no pane refresh.
- Do not parse or reformat read output. Raw bytes into the transcript. The transparency is the point.
- Do not modify PalanaCore. AT-01 built the boundary; if the surface needs something the Workbench does not offer, stop and surface rather than reaching past the API—that reach is exactly what this ho exists to forbid.
- Do not build any mutation verb or button. v1 is the four reads.

**Stop Condition**

If `RoutingConduit` does not route the read path to the local host and no clean per-host routing conduit is available, stop and surface—do not run every read over ssh (a local `df` must not need the wire). If terminal focus fights the key monitor the way KeysPanel did, follow the by-window-identity precedent or surface rather than adding a third event path.
