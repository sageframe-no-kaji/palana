# Ho-09-AT-02 — The field view overlay in the Surface

**Goal**

Build the summonable field view: a centered in-window overlay card rendering `FieldOutline`, summoned by `f`, navigated by the overlay's own five-key grammar, pointing the focused pane on Enter, re-probing one host on `r`. Depends on Ho-09-AT-01 (`FieldOutline`, `Field.allFacts()`) being in the tree.

**Context**

ho-09 Decisions 1–4 govern. Presentation is the help card's machinery—a SwiftUI `.overlay` on the panes in `SurfaceView`, not a window, not a sheet (the keys card is an `NSPanel` because it floats while the operator works elsewhere—the field view never does). The key monitor already routes by mode branches in `PalanaSession.handle(_:)`—`gotoTarget`, `pathEditing`, `helpVisible`, `operation.panelShowing`—read that method first (`Sources/Palana/PalanaSession.swift:149`) and match its shape. `Grammar.token(for:)` is the event-to-token mapping—reuse it, do not read `NSEvent` characters yourself. Theme tokens live in `Sources/Palana/Theme.swift`. Render facts in the pane's manners: `Theme.inkFaint` for secondary facts, `Theme.alarm` for unreachable detail, `Theme.accent` for the cursor row (match how `PaneView` and `HelpOverlay` compose).

**Files**

- Create: `Sources/Palana/FieldOverlay.swift` (the card view + `FieldViewModel`)
- Modify: `Sources/Palana/PalanaSession.swift` (state, summon, the `fieldVisible` branch)

**Required Changes**

1. **`FieldViewModel`** in `Sources/Palana/FieldOverlay.swift` — `@MainActor @Observable final class`, the `PaneModel` pattern: thin, delegating every transition to the core value.

   - Holds `outline: FieldOutline?`, `probing: Set<String>` (hosts with a probe in flight), `probeErrors: [String: String]` (detail per host from a thrown probe), and whatever the view needs—keep it minimal.
   - `summon()` — reads `engine.field.allFacts()` and `hosts` (the session's list, local first) in a `Task`, builds the outline, shows the card. The render is a cache read—no wire.
   - `reprobe()` — the host under the cursor. No-op for the local host and while that host is already probing. Marks probing, `try await engine.field.discover(host)`, on throw records the error detail, either way refreshes via `allFacts()` and `outline.update(facts:)`. The card updates in place—"probing…" on that row while in flight.
   - Cursor/expand/collapse methods forwarding to the outline's mutating transitions.

2. **Session state and summon** in `Sources/Palana/PalanaSession.swift`:

   - `fieldVisible` state (or the view model's presence—your call, match how `helpVisible` is held).
   - Summon on token `"f"`: in the main grammar path only—not while `gotoTarget != nil`, not while `pathEditing`, not while `operation.panelShowing` (the panel's keys are the panel's). Summoning while `helpVisible` hides help first—help and field never show together (the keys-card precedent at `SurfaceView.swift:27`).
   - **The `fieldVisible` branch** in `handle(_:)`, checked before the recognizer the way `helpVisible` is: `j`/`k`/`down`/`up` cursor, `l`/`h`/`right`/`left` expand/collapse, `r` re-probe, `f` and `esc` dismiss, `tab` passes through to its one meaning (switch pane—the pointing target follows the focus dot), `return` resolves `outline.pointing()`—when non-nil, `point(focusedSide, host:, path:)` and dismiss, when nil, nothing. Swallow every other token—the overlay is modal to the grammar. `cmd-` chords the grammar has no opinion about pass through to the system (match the help branch).

3. **The card** in `Sources/Palana/FieldOverlay.swift` — `FieldOverlay: View`, rendered from the view model's outline:

   - Centered card over the panes, panes dimmed beneath (match the help card's ground/shadow manners so the two read as siblings). Generous but bounded—it is a consultation, not a takeover.
   - Host line: alias, then quietly—reachability verdict with `FieldAge.describe` age ("reachable · 3h ago"), flavor, `zfs`/`rsync` presence tokens. Never-visited renders the alias plain with "never visited" in `inkFaint`. Unreachable renders its detail in `Theme.alarm`, quiet weight. Probing renders "probing…" in place of the verdict. Local renders as "this machine", no age, no probe.
   - Dataset line, indented under its expanded host: name, mountpoint in `inkFaint`. Non-pointable (unmounted, `legacy`, `none`) renders dimmer still—visible, not a destination.
   - Cursor row wears the accent wash, the pane's selection manner.
   - Footer, one quiet line: the five keys named (`↵ point · l expand · r re-probe · esc dismiss`—final phrasing yours, keep it short) and the `~` truth: "~ is the remote user's home". ho-09 carries this sentence deliberately—the practitioner read `/config` as a bug twice.

4. **`SurfaceView`** — the `.overlay` gated on the field's visibility, beside the help overlay's (`Sources/Palana/SurfaceView.swift:15`). Summoning help while the field is up dismisses the field (mutual exclusion, both directions).

5. **The `?` card** in `Sources/Palana/HelpOverlay.swift` gains one row for `f` — "field view" — wherever the vocabulary lists single keys. Match the card's existing grouping.

**Do Not**

- Do not build a second window or `NSPanel`—Decision 1 chose the in-window overlay deliberately.
- Do not add type-to-filter, a probe-all verb, host filtering, or services rows—out of scope by name in the ho.
- Do not touch `Grammar.bindings`—`f` routes through the mode branch, not the recognizer, the way the panel's keys do.
- Do not poll or auto-refresh—`r` under the operator's finger is the only re-probe (Decision 4).

**Stop Condition**

If the overlay cannot honestly render "instantly" from the cache—if `allFacts()` or outline building visibly lags the summon—stop and surface before adding spinners or async ceremony to the summon path. The two-second commitment is the design's, and a slow cache read means something upstream is wrong.
