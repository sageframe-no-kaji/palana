**Required Changes**

1. **`ZFSMutation`** ‚Äî the enum from Decision 1 (`createDataset`, `destroyDataset`, `renameDataset`, `snapshot`, `destroySnapshot`, `rollback`, `setMountpoint`, `clearMountpoint`), `Sendable, Equatable, Codable`.

2. **`PlanOperation.zfs`**, **`PlanRequest.zfs: ZFSMutation?`** (nil default), **`Classification.zfsMutation`**. `PlanRequest` for a ZFS mutation carries `operation: .zfs`, a `source` locus naming the host, empty `entries`, nil `destination`, and the `zfs` payload.

3. **`classify()`** routes `.zfs` ‚Üí `.zfsMutation`. **`transport()`** returns `.local` for `.zfsMutation`. **`compose()`** dispatches `.zfsMutation` ‚Üí `composeZFSMutation`.

4. **`composeZFSMutation(request:)`** switches on `request.zfs` and builds `[PlanStep]` per Decision 3‚Äîthe `zfs` command step then the `.verify` read step, all on `request.source.host`, all `ShellQuote`d:
   - createDataset: `zfs create <name>` (+ `-o mountpoint=<path>` when non-nil) ; verify `zfs list -H -o name -- <name>`
   - destroyDataset: `zfs destroy <name>` / `zfs destroy -r <name>` (recursive stated) ; verify the name lists no longer
   - renameDataset: `zfs rename <old> <new>` ; verify new present and old absent
   - snapshot: `zfs snapshot <dataset>@<name>` / `-r` ; verify the snapshot lists (`zfs list -H -o name -t snapshot -- <dataset>@<name>`)
   - destroySnapshot: `zfs destroy <dataset>@<name>` ; verify absent
   - rollback: `zfs rollback <dataset>@<name>` ; verify (snapshot still lists)
   - setMountpoint: `zfs set mountpoint=<path> <dataset>` ; verify `zfs get -H -o value mountpoint -- <dataset>` (the verify reads the value; the exact assertion mechanism matches how existing `.verify` steps read state)
   - clearMountpoint: `zfs inherit mountpoint <dataset>` ; verify
   No `sudo`. Delegated. Use `--` before names where the tool accepts it, matching how existing composes guard.

5. **Validation** (in `validate()` or a ZFS-specific guard the engine calls for `.zfs`): a `.zfs` request requires a non-nil `zfs` payload; dataset/snapshot names non-empty; snapshot names contain no `@` (the composer joins it); a rename's `from` and `to` non-empty and differing. Refuse malformed shapes the way the engine already throws (read how `.rename`/`.create` refuse).
