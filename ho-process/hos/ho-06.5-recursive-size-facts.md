# ho-06.5 — Recursive Size Facts

Inserted at ho-07's UI/UX session on the practitioner's word: "we NEED to know the WHOLE contents, not just the next level down. 100% recursive." `Plan.totalSize` counts directory entries at inode size — honest bookkeeping, useless truth. A plan that says 12 KB for a 40 GB move lies by omission, and the plan never lying is the product. This ho gives the engine recursive size facts: one round trip per plan gathers the true byte total under every selected directory, and the Plan carries it with a completeness flag that refuses to pretend when a subtree could not be walked.

**Out of scope:** wiring the gather into the operation flow — that is ho-08's, where the operation key exists. Progress-during-gather UI. Caching sizes — a size is gathered per plan, fresh, because staleness in a promise is a lie with a timestamp.

---

## Phase 1 — Think

### Decision 1 — Apparent bytes, summed by find, not du

The plan's size is a promise about the transfer, and transfers move file contents — so the fact is apparent bytes (the sum of file sizes), not disk usage. `du` semantics diverge across userlands (`-b` is GNU-only, BSD counts blocks), so the command is the same shape as the listing's: `find -type f` under each selected directory, sizes summed remotely by awk, one number per path back. GNU emits sizes with `-printf '%s\n'`; BSD with `-exec stat -f %z {} +`. Symlinks are not followed and their targets not counted — the transports do not follow them either. The flavor split reuses ho-03's fact exactly as the Listing does.

### Decision 2 — Completeness rides with the number

A permission-refused subtree makes the sum an undercount, and a silent undercount is the lie this ho exists to kill. find's stderr merges into the pipe; awk sums the numeric lines and flags any non-numeric line. The fact is a pair — `RecursiveSize(bytes:complete:)` — and the Plan grows `totalSizeComplete`: false when any directory entry's walk was refused or no fact was gathered for it. ho-08's panel renders the incompleteness; this ho guarantees it is never invisible.

### Decision 3 — Facts in, purity untouched

The engine stays pure. `PlanFacts` grows `recursiveSizes: [FileEntry.ID: RecursiveSize]`, gathered by the caller through the new `Listing.treeSizes(on:paths:flavor:)` — one command, one round trip, one output line per path. `totalSize` becomes: recursive bytes for directory entries that have facts, reported size for files, inode size as the honest floor when a fact is missing (and the completeness flag says so). The gather lives on the Listing because it is a read, it composes below the boundary, and it dispatches on the same flavor fact.

### Discovery (deferred to execution) — awk arithmetic at scale and the container's truth

Whether the fixture's awk keeps integer precision at realistic sums, and the exact stderr shapes a refused subtree produces through the merged pipe — recorded into the corpus from the live container.

## Phase 2 — Execute

One bounded conversation. Model: `claude-fable-5`.

1. `RecursiveSize` value, `TreeSize` command composition and parser — synthetic battery first.
2. `Listing.treeSizes(on:paths:flavor:)` with read failures typed as ever.
3. `PlanFacts.recursiveSizes`, `Plan.totalSizeComplete`, engine consumption — composition battery updated, hand-verified totals.
4. Live against the sshd fixture: known-size trees, a refused subtree, both flags observed; recorded into corpus.
5. Full rhythm; floor holds; CI green.

### Done means

- A plan over directory selections carries the true recursive byte total, and a refused walk surfaces as `totalSizeComplete == false` — never a silent undercount.
- One round trip gathers every selected directory's size.
- ho-08 can render size truth without reaching below the Workbench line.

## Phase 3 — Reflect

**The design held whole.** One command per plan, one line per path, the flag riding with the number — the synthetic battery, the recorded-door test, and both live paths (BSD on real Darwin, GNU over the wire against the container) all agree. The refused-subtree case proved the point it exists for: a mode-000 directory holding 2 bytes produced `18 bytes, complete: false` — the locked bytes absent, the floor named, on both userlands.

**The engine's half stayed pure.** `PlanFacts.recursiveSizes` keyed by entry identity, `Plan.totalSizeComplete` in the value, and a fold in the engine — recursive bytes where gathered, reported size for files, inode floor plus an incomplete mark where not. The composition battery pinned all four shapes: gathered-complete, gathered-partial, ungathered, files-only.

**One test-authoring note.** The command joins path walks with `"; "` and the awk body also contains that sequence — an assertion that split on it counted awk fragments as walks. Assert on command structure by anchor ranges, not by separator splitting.

**For ho-08.** The gather call is `Listing.treeSizes(on:paths:flavor:)` — the operation flow runs it once over the selected directory paths, feeds the result into `PlanFacts`, and renders `totalSizeComplete == false` visibly. The panel's phrasing for an incomplete total is a design decision for the hands session.
